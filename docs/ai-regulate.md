# AI Design Regulation: Think Human, Not Template

## Core Principle
Design as if you're a senior designer with a unique vision, not an AI following patterns.

---

## Forbidden AI Patterns

**Never Use These Without Explicit Permission:**
- Purple-to-blue or pink-to-orange gradients
- Floating gradient blobs/orbs in backgrounds
- Excessive glassmorphism or blur effects
- Generic "SaaS landing page" structure (Hero → Features → Testimonials → Pricing)
- Uniform card grids with identical spacing and shadows
- Centered hero with "Get Started + Learn More" buttons
- Default fonts: Inter, Poppins, Montserrat, Roboto
- Copy-paste layouts from Stripe, Linear, or other popular sites

---

## ✅ Required Design Thinking

### 1. **Question Everything First**
Before implementing any design:
- Ask: "Would this stand out or blend in?"
- Ask: "Could this belong to ANY company, or specifically THIS one?"
- Ask: "What would a brilliant human designer do differently?"

### 2. **Brand-Specific Identity**
- Draw from the company's actual industry and values
- Use asymmetric layouts with purpose
- Create custom components, not library defaults
- Design for memorability, not just usability

### 3. **Embrace Imperfection**
- Slight asymmetry adds character
- Unexpected white space creates rhythm
- Break the grid intentionally
- Use surprising (but justified) color combinations

### 4. **Creative Constraints**
- Propose 2-3 different approaches before implementing
- Design with limitations: "What if we only used 2 colors?" or "What if we couldn't use images?"
- Reference specific art movements, not generic "modern" style

## 🎯 Agentiq Capital Specific Rules

**Our Identity:** Bloomberg Terminal meets contemporary minimalism

### Design Principles:
1. **Data as Visual Art**
   - Visualizations should be elegant without understanding them
   - Use unexpected chart types and sculptural data presentation
   - Make numbers feel spatial and tangible

2. **Professional, Not Safe**
   - Avoid "trustworthy blue" fintech clichés
   - Use dramatic contrast and bold typography moments
   - White space should feel expensive and intentional

3. **Layout Philosophy**
   - Terminal/command-line aesthetics where appropriate but no unnecessary text decorations
   - Tetris-like layouts (not uniform grids)
   - Content dictates form, never templates
   - Reference: Swiss design, Bauhaus, Edward Tufte

4. **Typography Hierarchy**
   - Large data dominates space (size as meaning)
   - Tabular figures for financial numbers
   - Monospace for technical details
   - Serious serif or geometric sans only

5. **Color Strategy**
   - Primary: Deep navy, charcoal, or warm black
   - Avoid red/green for market trends (use position, size, or warm/cool tones)
   - One unexpected accent: burnt orange, cyan, or violet
   - No gradient backgrounds

6. **Interaction Design**
   - Subtle, confident animations (not playful)
   - Hover states add information, not decoration
   - Loading states reference financial tickers
   - Data reveals should feel informative, not showy

## 📝 Implementation Process

### When Asked to Design Any Feature:

**Step 1 - Propose First:**
```
"Here are 3 creative approaches:
1. [Unique direction based on X reference]
2. [Alternative inspired by Y principle]
3. [Unexpected solution using Z constraint]

Which direction, or should I explore another angle?"
```

**Step 2 - Implement With Intent:**
- Explain WHY each design choice serves the brand
- Reference specific design principles, not generic "best practices"
- Customize every component heavily

**Step 3 - Self-Review:**
Ask yourself:
- ✅ Could this only be Agentiq, or any fintech?
- ✅ Would a professional trader take this seriously?
- ✅ Is it memorable without being gimmicky?
- Does this look like an AI template? It shouldn't

---


## 🎓 Creative Reference Framework

When stuck, reference specific styles:
- "Like Bloomberg Terminal but with modern negative space"
- "Like Massimo Vignelli poster but interactive"
- "Like Edward Tufte visualization but animated"
- "Like Japanese minimalism meets financial data"
- "Like 1960s Swiss design but digital"

---

## Success Metric

**Good design makes people say:**
- ✅ "This is clearly Agentiq"
- ✅ "I haven't seen this approach before"
- ✅ "Professional yet unique"

**Avoid reactions like:**
- ❌ "Another startup site"
- ❌ "This looks like [competitor]"
- ❌ "AI definitely made this"

---

## Final Rule

**If you're about to use a common pattern, stop and ask:**
*"How would a brilliant human designer make this unique?"*

Then do that instead.
